# dtr2epf

GUI-приложение на Rust для сборки модуля внешней обработки 1С из исходников Datareon Platform.

Интерфейс использует CPU software renderer и не требует OpenGL, DirectX 12 или дискретной видеокарты.

## Возможности

- загружает адаптеры из `Adapters` с `Config.ConnectorType = "_1C"`;
- загружает `Subscription1C` и `Function1C` из `Metadata/Handlers`;
- восстанавливает дерево каталогов по `FolderId` и `EntityId`;
- переносит выбор адаптера на связанные элементы из `Config.HandlersList`;
- поддерживает групповой выбор, поиск и сортировку адаптеров/дерева;
- читает код из файлов, указанных в поле `Code` (`... [Code].ext`);
- формирует `ObjectModule.bsl`, разделяя входящие, исходящие обработчики и функции по областям;
- копирует сформированный модуль в системный буфер обмена кнопкой или сочетанием `Ctrl+C`;
- проверяет пустой выбор, отсутствующий код, дубли имён и обязательные плейсхолдеры;
- позволяет редактировать, импортировать и экспортировать шаблоны.

Шаблоны в каталоге [`templates`](templates) подключаются через `include_str!` и компилируются в исполняемый файл. Для входящих `FromPlatform` и исходящих `ToPlatform` обработчиков используются разные шаблоны. Кнопка «Восстановить встроенные» всегда возвращает версию шаблонов из бинарника.

## Запуск и сборка

```powershell
cargo run
cargo build --release
```

Обычный запуск открывает приложение без выбранного каталога. Чтобы сразу загрузить определённые исходники, используйте ключ:

```powershell
dtr2epf.exe --source "D:\Datareon\project\src"
```

Исполняемый файл после release-сборки: `target\release\dtr2epf.exe`.

Запустить автоматические тесты:

```powershell
cargo test
```

Тесты создают изолированный временный пример структуры Datareon и проверяют загрузку JSON/кода, адаптеры, дерево папок, генерацию, шаблоны, аргументы запуска и сброс состояния GUI.

Подготовленны1 пример исходников для интеграционных тестов размещаются в [`tests/fixtures/datareon_sample`](tests/fixtures/datareon_sample).

Проверить загрузчик без открытия GUI:

```powershell
cargo run --features console -- --check "tests\fixtures\datareon_sample"
```

В Windows обычная GUI-сборка запускается без консольного окна. Фича
`console` включает консольную подсистему для диагностического режима `--check`.

## Генерация

1. Выберите каталог `src` конфигурации Datareon кнопкой «Обзор…» — дерево загрузится сразу. Путь также можно ввести вручную и нажать «Загрузить».
2. Выберите адаптеры или отдельные строки в дереве.
3. Выберите вариант генерации: «Для отладки» или «Для синтаксического контроля». По умолчанию используется вариант «Для отладки».
4. При необходимости измените шаблоны и имена областей.
5. Нажмите «Сформировать обработку» и сохраните `ObjectModule.bsl`.
6. Поместите модуль в исходники внешней обработки, соберите EPF и запустите требуемые синтаксические контроли/линтеры 1С.

Для вариантов генерации используются отдельные шаблоны входящих и исходящих обработчиков. Шаблоны синтаксического контроля находятся в каталоге [`templates/syntax_control`](templates/syntax_control).

Приложение формирует исходный BSL-модуль; сборка бинарного EPF остаётся за инструментами 1С/Designer/Vanessa/OScript, используемыми в конкретном проекте.

## Плейсхолдеры

Шаблон модуля использует `{{FROM_PLATFORM}}`, `{{TO_PLATFORM}}`, `{{FUNCTIONS}}`, `{{REGION_*}}`, `{{GENERATED_AT}}` и `{{SELECTED_COUNT}}`.

Шаблоны методов используют `{{NAME}}`, `{{ORIGINAL_NAME}}`, `{{PARAMETERS}}`, `{{CODE}}`, `{{TYPE}}`, `{{INTEGRATION}}`, `{{ENTITY_ID}}`, `{{SOURCE_PATH}}`, `{{SUBSCRIPTION_OBJECT}}` и `{{SUBSCRIPTION_OBJECT_MANAGER}}`.

`{{SUBSCRIPTION_OBJECT}}` и `{{SUBSCRIPTION_OBJECT_MANAGER}}` заполняются только для обработчиков `Subscription1C / ToPlatform`. Например, для `Справочник.Автомобили` значением менеджера будет `Справочники.Автомобили`.
